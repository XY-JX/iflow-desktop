# 本地开发规则

## 开发环境要求

### Node.js 版本
- 推荐版本：v20.x
- 最低要求：v18.x

### Rust 工具链
```bash
rustup default stable
```

## 代码规范

### TypeScript
- 使用严格模式
- 避免使用 `any` 类型
- 未使用的变量会显示警告
- 推荐使用 `const` 而非 `var`

### Vue 组件
- 组件名称可以使用单个单词
- Props 不需要默认值
- 不要直接修改 props

### 测试
- 使用 Vitest 测试框架
- 全局测试变量已配置（describe, it, expect 等）
- 测试文件命名：`*.test.ts`

## 本地命令

### 开发
```bash
npm run dev        # 启动开发服务器
npm run tauri dev  # 启动 Tauri 开发环境
```

### 构建
```bash
npm run build      # 构建前端
npm run tauri build # 构建桌面应用
```

### 测试
```bash
npm run test       # 运行测试
npm run test:watch # 监听模式运行测试
```

### 代码检查
```bash
npm run lint       # ESLint 检查
```

## 项目结构

```
iflow-desktop/
├── src/                    # 前端源代码
│   ├── components/        # Vue 组件
│   ├── stores/           # Pinia 状态管理
│   ├── types/            # TypeScript 类型定义
│   ├── __tests__/        # 测试文件
│   └── assets/           # 静态资源
├── src-tauri/            # Rust 后端代码
├── .github/workflows/    # CI/CD 配置
└── .iflow/              # 项目文档
```

## 提交规范

Commit message 格式：
```
<type>: <description>
```

Type 包括：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具配置

示例：
```
fix: 修复 MainLayout 响应式问题
feat: 添加文件浏览器功能
```

## 注意事项

1. **不要提交的文件**
   - `node_modules/`
   - `dist/`
   - `.env.local`
   - IDE 配置文件（`.vscode/*` 除外）

2. **环境变量**
   - 敏感信息使用 `.env.local` 文件
   - 不要将密钥提交到仓库

3. **跨平台开发**
   - Windows: 需要 MSVC 编译环境
   - macOS: 需要 Xcode Command Line Tools
